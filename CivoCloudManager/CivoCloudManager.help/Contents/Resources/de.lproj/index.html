Civo Cloud Manager Hilfe

Verwalten Sie Ihre Civo Cloud Infrastruktur direkt von Ihrem Mac.

Erste Schritte

Civo Cloud Manager wird als Menüleisten-App ausgeführt. Nach dem Start erscheint ein Schild-Symbol in der Menüleiste Ihres Macs.

  1. Klicken Sie auf das Schild-Symbol, um das Popover zu öffnen.
  2. Beim ersten Start öffnet sich der Einrichtungsassistent automatisch.
  3. Geben Sie Ihren Civo API Key ein (zu finden unter civo.com → Account → Security → API Keys).
  4. Wählen Sie Ihre Region (z.B. fra1, lon1, nyc1).
  5. Wählen Sie, welche Firewalls verwaltet werden sollen, und legen Sie Ports fest.
  6. Optional können Sie Beim Anmelden starten aktivieren.

Menüleiste

Das Menüleisten-Popover bietet schnellen Zugriff auf die Firewall-Verwaltung:

SymbolBedeutung
Schild (grün)Alle Firewalls geschlossen
Schild (gelb)Einige Firewalls für Ihre IP geöffnet
Schild (rot)Einrichtung nicht abgeschlossen
  • Öffnen — erstellt eine Firewall-Regel, die Ihre aktuelle öffentliche IP auf dem konfigurierten Port zulässt.
  • Schließen — entfernt die Regel.
  • Alle öffnen / Alle schließen — Massenaktionen für alle verwalteten Firewalls.
  • Dashboard — öffnet das vollständige Verwaltungsfenster.
  • Einstellungen — öffnet den Einrichtungsassistenten erneut.
  • Aktualisieren — prüft den Status manuell erneut.

Ihre öffentliche IP wird automatisch über ipify.org mit Fallback-Anbietern erkannt. Die App verwaltet nur Regeln, die sie selbst erstellt hat — gekennzeichnet mit Ihrem Hostnamen.

Dashboard

Das Dashboard zeigt Ressourcenanzahlen und Kontingentnutzung. Klicken Sie auf eine Ressourcenkarte, um direkt zu diesem Bereich zu navigieren.

Kontingent-Anzeigen

Kreisförmige Anzeigen zeigen die Nutzung im Vergleich zum Limit für alle Kontokontingente (RAM und DB RAM werden in GB angezeigt).

Änderung anfordern

Klicken Sie auf Änderung anfordern neben „Kontingentnutzung“, um ein Formular mit Steppern für alle Kontingentlimits zu öffnen. Sendet eine Änderungsanfrage über die Civo API.

Compute

Instanzen

Compute-Instanzen anzeigen, erstellen und löschen. Klicken Sie auf eine Instanz, um Details zu sehen:

  • CPU, RAM und Festplattenspezifikationen
  • Öffentliche und private IP-Adressen
  • Initiales Root-Passwort (Ein-/Ausblenden-Schalter)
  • Tags, Region, Erstellungsdatum
  • Netzwerk- und Firewall-Zuordnungen

Zum Erstellen: Klicken Sie auf + in der Symbolleiste. Füllen Sie Hostname, Größe, Disk-Image, Netzwerk, Firewall, SSH Key und Tags aus.

SSH Keys

SSH Keys für den Instanzzugriff verwalten. Um einen Key hinzuzufügen: Klicken Sie auf +, geben Sie einen Namen ein und fügen Sie Ihren öffentlichen Schlüssel ein.

Kubernetes

Cluster

Kubernetes-Cluster anzeigen, erstellen und löschen. Klicken Sie auf einen Cluster, um die Detailansicht zu sehen mit:

  • Statistiken — Nodes, Pools, Integritätsstatus, installierte Apps auf einen Blick.
  • Cluster-Info — API-Endpunkt, Master-IP, DNS, CNI-Plugin, Node-Größe, Erstellungsdatum.
  • Bedingungen — ControlPlaneReady, WorkerNodesReady, ClusterVersionSync mit Integritätsindikatoren.
  • Node Pools — klicken, um Pool-ID, Größe, Node-Anzahl, öffentliche IPs, Instanznamen, Labels und Taints anzuzeigen.
  • Labels — klicken Sie auf Bearbeiten, um Labels eines Node Pools hinzuzufügen, zu entfernen oder zu ändern.
  • Installierte Anwendungen — cert-manager, ingress-nginx usw.
  • Kubeconfig speichern — Symbolleisten-Schaltfläche zum Export der Kubeconfig als .yaml-Datei.

Zum Erstellen: Klicken Sie auf +. Konfigurieren Sie Name, CNI-Plugin, Node-Größe, Node-Anzahl, Netzwerk und Marketplace-Apps.

Node-Details (K8s API)

Klicken Sie auf einen Node-Namen in einem Node Pool, um sich direkt mit der Kubernetes API zu verbinden und detaillierte Node-Informationen zu sehen:

  • Ressourcenkarten — CPU, Memory und Pods mit Kapazität vs. verfügbaren Werten.
  • Bedingungen — Ready, MemoryPressure, DiskPressure, PIDPressure mit Integritätsindikatoren.
  • Adressen — InternalIP und Hostname.
  • Systeminformationen — Betriebssystem, Architektur, Container-Runtime, Kubelet-Version.
  • Pods auf diesem Node anzeigen — Schaltfläche zum Anzeigen aller Pods, die auf diesem Node laufen.

Die App lädt die Kubeconfig von der Civo API herunter, extrahiert die Zertifikate und verbindet sich direkt mit der Kubernetes API über Client-Zertifikat-Authentifizierung. Kein kubectl oder andere externe Tools erforderlich.

Pod-Liste

Klicken Sie in der Node-Detailansicht auf Pods auf diesem Node anzeigen, um alle Pods auf diesem Node zu sehen. Jeder Pod zeigt:

  • Status-Badge (Running, Pending, Failed usw.)
  • Namespace
  • Anzahl bereiter Container
  • Anzahl der Neustarts

Pod-Logs

Klicken Sie auf einen Pod in der Pod-Liste, um seine Logs anzuzeigen:

  • Scrollbare Logausgabe in Monospace-Schrift
  • Auto-Scroll-Schalter — scrollt automatisch zu den neuesten Logeinträgen
  • Aktualisieren-Schaltfläche — lädt die neuesten Logs neu
  • Pod-Header mit Name, Namespace und Status

Netzwerk

Netzwerke

Virtuelle Netzwerke anzeigen und erstellen. Das Standardnetzwerk kann nicht gelöscht werden. Klicken Sie auf +, um eines mit einem Label und optionalem CIDR zu erstellen.

Firewalls

Klicken Sie auf eine Firewall, um alle Regeln anzuzeigen. Jede Regel zeigt:

  • Protokoll, Portbereich, CIDR
  • Richtung (Ingress/Egress) und Aktion (Erlauben/Ablehnen) als farbige Badges
  • Label

Klicken Sie auf + in der Regeldetailansicht, um eine neue Regel hinzuzufügen. Löschen Sie Regeln über das Rechtsklick-Kontextmenü (erfordert die Eingabe des Regelnamens).

Load Balancer

Load-Balancer-Details anzeigen, einschließlich Algorithmus, öffentliche/private IPs, Traffic-Richtlinie und Backend-Liste. Löschen über das Kontextmenü.

Domains & DNS

Erweitern Sie eine Domain, um alle DNS-Einträge zu sehen. Jeder Eintrag zeigt Typ, Name, Wert und TTL.

  • Eintrag hinzufügen — klicken Sie auf die Schaltfläche „Eintrag hinzufügen“ innerhalb der Domain.
  • Eintrag bearbeiten — Rechtsklick auf einen Eintrag und „Bearbeiten“ auswählen.
  • Eintrag löschen — Rechtsklick und „Löschen“ auswählen.

Speicher & Daten

Datenbanken

Klicken Sie auf eine Datenbank, um Verbindungsdetails (Host, Port, DNS-Eintrag, private IP), Konfiguration (Software, Version, Größe, Nodes) und Netzwerk-/Firewall-Zuordnungen zu sehen.

Zum Erstellen: Klicken Sie auf +. Wählen Sie PostgreSQL oder MySQL, Version, Größe, Nodes, Netzwerk und Firewall.

Volumes

Klicken Sie auf ein Volume, um den Anhangsstatus (an welche Instanz oder welchen Cluster es angehängt ist), Mountpoint, Größe, Boot-Status, Region und Erstellungsdatum zu sehen.

Die Liste zeigt, ob jedes Volume „an Instanz angehängt“, „an Cluster angehängt“ oder „verfügbar“ ist.

Verfügbare bereinigen — Symbolleisten-Schaltfläche zum Massenlöschen aller Volumes mit dem Status „verfügbar“. Erfordert die Eingabe von DELETE ALL DATA zur Bestätigung.

Object Stores

Object Stores mit Endpunkt und maximaler Größe anzeigen. Rechtsklick für:

  • Zugangsdaten anzeigen — zeigt Access Key ID und Secret Access Key an (zum Kopieren auswählbar).
  • Löschen — entfernt den Object Store dauerhaft.

Konto

Regionen

Alle verfügbaren Civo-Regionen anzeigen. Klicken Sie auf Verwenden, um Ihre aktive Region zu wechseln. Alle nachfolgenden API-Aufrufe werden die ausgewählte Region verwenden.

Über

Zeigt die App-Version, Build-Nummer, Entwicklerinformationen und Links zu Datenschutzrichtlinie und Nutzungsbedingungen.

Ressourcen löschen

Alle Löschvorgänge erfordern die Eingabe des exakten Ressourcennamens zur Bestätigung. Dies verhindert das versehentliche Löschen kritischer Cloud-Infrastruktur.

Zum Löschen: Rechtsklick auf eine Ressource und Löschen auswählen. Ein Bestätigungsdialog erscheint, in dem Sie den Namen exakt wie angezeigt eingeben müssen. Die Lösch-Schaltfläche wird erst aktiv, wenn der Name übereinstimmt.

Vollzugriff

Die Menüleisten-Firewall-Verwaltung ist kostenlos. Das vollständige Dashboard mit allen Ressourcenverwaltungsfunktionen erfordert einen einmaligen Kauf von Vollzugriff (14,99 $).

  • Kauf wiederherstellen — wenn Sie zuvor auf einem anderen Gerät gekauft haben.
  • Code einlösen — für Apple-Angebotscodes.
  • Familienfreigabe — aktiviert.

Tipps

  • Alle Textfelder mit IPs, Endpunkten und Zugangsdaten unterstützen Textauswahl zum einfachen Kopieren.
  • Die App aktualisiert den Firewall-Status automatisch alle 60 Sekunden.
  • Dashboard-Ressourcenkarten sind anklickbar — navigieren Sie mit einem Klick zu jedem Bereich.
  • Von dieser App erstellte Firewall-Regeln sind als civo-cloud-<hostname>-<firewall> gekennzeichnet und nur diese Regeln werden verwaltet.
  • Der Kubernetes-Drill-Down-Ablauf: Cluster-Liste → Cluster-Detail → Node-Detail → Pod-Liste → Pod-Logs.
