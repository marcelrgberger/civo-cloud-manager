Civo Cloud Manager Help

Manage your Civo Cloud infrastructure from your Mac.

Getting Started

Civo Cloud Manager runs as a menu bar app. After launching, a shield icon appears in your Mac's menu bar.

  1. Click the shield icon to open the popover.
  2. If this is your first launch, the Setup Wizard opens automatically.
  3. Enter your Civo API Key (find it at civo.com → Account → Security → API Keys).
  4. Select your region (e.g. fra1, lon1, nyc1).
  5. Choose which firewalls to manage and set ports.
  6. Optionally enable Launch at Login.

Menu Bar

The menu bar popover provides quick-access firewall management:

IconMeaning
Shield (green)All firewalls closed
Shield (yellow)Some firewalls open for your IP
Shield (red)Setup not complete
  • Open — creates a firewall rule allowing your current public IP on the configured port.
  • Close — removes the rule.
  • Open All / Close All — bulk actions for all managed firewalls.
  • Dashboard — opens the full management window.
  • Settings — re-opens the setup wizard.
  • Refresh — manually re-checks status.

Your public IP is auto-detected via ipify.org with fallback providers. The app only manages rules it created — labeled with your hostname.

Dashboard

The Dashboard shows resource counts and quota usage. Click any resource card to navigate directly to that section.

Quota Gauges

Circular gauges show usage vs. limit for all account quotas (RAM and DB RAM displayed in GB).

Request Quota Change

Click Request Change next to "Quota Usage" to open a form with steppers for all quota limits. Submits a change request via the Civo API.

Compute

Instances

View, create, and delete compute instances. Click an instance to see details:

  • CPU, RAM, and Disk specifications
  • Public and private IP addresses
  • Initial root password (show/hide toggle)
  • Tags, region, creation date
  • Network and firewall associations

To create: click + in the toolbar. Fill in hostname, size, disk image, network, firewall, SSH key, and tags.

SSH Keys

Manage SSH keys for instance access. To add a key: click +, enter a name, and paste your public key.

Kubernetes

Clusters

View, create, and delete Kubernetes clusters. Click a cluster to see its detail view with:

  • Stats — Nodes, Pools, Health status, installed Apps at a glance.
  • Cluster Info — API endpoint, master IP, DNS, CNI plugin, node size, creation date.
  • Conditions — ControlPlaneReady, WorkerNodesReady, ClusterVersionSync with health indicators.
  • Node Pools — click to expand and see pool ID, size, node count, public IPs, instance names, labels, and taints.
  • Labels — click Edit to add, remove, or modify labels on a node pool.
  • Installed Applications — cert-manager, ingress-nginx, etc.

To create: click +. Configure name, CNI plugin, node size, node count, network, and marketplace apps.

Networking

Networks

View and create virtual networks. The default network cannot be deleted. Click + to create with a label and optional CIDR.

Firewalls

Click a firewall to see all its rules. Each rule shows:

  • Protocol, port range, CIDR
  • Direction (ingress/egress) and action (allow/deny) as colored badges
  • Label

Click + in the rule detail view to add a new rule. Delete rules via right-click context menu (requires typing the rule name).

Load Balancers

View load balancer details including algorithm, public/private IPs, traffic policy, and backend list. Delete via context menu.

Domains & DNS

Expand a domain to see all DNS records. Each record shows type, name, value, and TTL.

  • Add Record — click the "Add Record" button inside the domain.
  • Edit Record — right-click a record and select "Edit".
  • Delete Record — right-click and select "Delete".

Storage & Data

Databases

Click a database to see connection details (host, port, DNS entry, private IP), configuration (software, version, size, nodes), and network/firewall associations.

To create: click +. Choose PostgreSQL or MySQL, version, size, nodes, network, and firewall.

Volumes

Click a volume to see attachment status (which instance or cluster it's attached to), mountpoint, size, bootable status, region, and creation date.

The list shows whether each volume is "attached to instance", "attached to cluster", or "available".

Cleanup Available — toolbar button to bulk-delete all volumes with status "available". Requires typing DELETE ALL DATA to confirm.

Object Stores

View object stores with endpoint and max size. Right-click to:

  • Show Credentials — displays access key ID and secret access key (selectable for copying).
  • Delete — permanently removes the object store.

Account

Regions

View all available Civo regions. Click Use to switch your active region. All subsequent API calls will target the selected region.

About

Shows the app version, build number, developer information, and links to Privacy Policy and Terms of Use.

Deleting Resources

All delete operations require you to type the exact resource name to confirm. This prevents accidental deletion of critical cloud infrastructure.

To delete: right-click a resource and select Delete. A confirmation sheet appears where you must type the name exactly as shown. The delete button only becomes active when the name matches.

Full Access

The menu bar firewall management is free. The full dashboard with all resource management requires a one-time purchase of Full Access ($14.99).

  • Restore Purchase — if you previously purchased on another device.
  • Redeem Code — for Apple offer codes.
  • Family Sharing — enabled.

Tips

  • All text fields with IPs, endpoints, and credentials support text selection for easy copying.
  • The app auto-refreshes firewall status every 60 seconds.
  • Dashboard resource cards are clickable — navigate to any section with one click.
  • Firewall rules created by this app are labeled civo-cloud-<hostname>-<firewall> and only this app's rules are managed.
