Ayuda de Civo Cloud Manager

Gestione su infraestructura de Civo Cloud desde su Mac.

Primeros pasos

Civo Cloud Manager se ejecuta como una app en la barra de menús. Tras el inicio, aparece un icono de escudo en la barra de menús de su Mac.

  1. Haga clic en el icono del escudo para abrir el popover.
  2. Si es la primera vez que lo abre, el Asistente de configuración se abre automáticamente.
  3. Introduzca su Civo API Key (encúentrelo en civo.com → Account → Security → API Keys).
  4. Seleccione su región (p. ej. fra1, lon1, nyc1).
  5. Elija qué firewalls gestionar y configure los puertos.
  6. Opcionalmente, active Iniciar al iniciar sesión.

Barra de menús

El popover de la barra de menús proporciona gestión rápida de firewalls:

IconoSignificado
Escudo (verde)Todos los firewalls cerrados
Escudo (amarillo)Algunos firewalls abiertos para su IP
Escudo (rojo)Configuración no completada
  • Abrir — crea una regla de firewall que permite su IP pública actual en el puerto configurado.
  • Cerrar — elimina la regla.
  • Abrir todos / Cerrar todos — acciones masivas para todos los firewalls gestionados.
  • Dashboard — abre la ventana de gestión completa.
  • Ajustes — vuelve a abrir el asistente de configuración.
  • Actualizar — vuelve a verificar el estado manualmente.

Su IP pública se detecta automáticamente a través de ipify.org con proveedores de respaldo. La app solo gestiona las reglas que ella misma creó — etiquetadas con su nombre de host.

Dashboard

El Dashboard muestra el recuento de recursos y el uso de cuotas. Haga clic en cualquier tarjeta de recurso para navegar directamente a esa sección.

Indicadores de cuota

Los indicadores circulares muestran el uso frente al límite para todas las cuotas de la cuenta (RAM y DB RAM se muestran en GB).

Solicitar cambio de cuota

Haga clic en Solicitar cambio junto a «Uso de cuota» para abrir un formulario con steppers para todos los límites de cuota. Envía una solicitud de cambio a través de la Civo API.

Compute

Instancias

Ver, crear y eliminar instancias de cómputo. Haga clic en una instancia para ver los detalles:

  • Especificaciones de CPU, RAM y disco
  • Direcciones IP públicas y privadas
  • Contraseña root inicial (botón mostrar/ocultar)
  • Tags, región, fecha de creación
  • Asociaciones de red y firewall

Para crear: haga clic en + en la barra de herramientas. Complete el hostname, tamaño, imagen de disco, red, firewall, SSH Key y tags.

SSH Keys

Gestione SSH Keys para el acceso a instancias. Para añadir una clave: haga clic en +, introduzca un nombre y pegue su clave pública.

Kubernetes

Clusters

Ver, crear y eliminar clusters de Kubernetes. Haga clic en un cluster para ver su vista de detalle con:

  • Estadísticas — Nodes, Pools, estado de salud, aplicaciones instaladas de un vistazo.
  • Info del cluster — endpoint de API, IP maestra, DNS, plugin CNI, tamaño de node, fecha de creación.
  • Condiciones — ControlPlaneReady, WorkerNodesReady, ClusterVersionSync con indicadores de salud.
  • Node Pools — haga clic para expandir y ver ID del pool, tamaño, número de nodes, IPs públicas, nombres de instancia, labels y taints.
  • Labels — haga clic en Editar para añadir, eliminar o modificar labels de un node pool.
  • Aplicaciones instaladas — cert-manager, ingress-nginx, etc.
  • Guardar Kubeconfig — botón en la barra de herramientas para exportar la kubeconfig como archivo .yaml.

Para crear: haga clic en +. Configure nombre, plugin CNI, tamaño de node, número de nodes, red y apps del marketplace.

Detalles del Node (K8s API)

Haga clic en el nombre de un node en un node pool para conectarse directamente a la API de Kubernetes y ver información detallada del node:

  • Tarjetas de recursos — CPU, Memory y Pods mostrando capacidad vs. valores asignables.
  • Condiciones — Ready, MemoryPressure, DiskPressure, PIDPressure con indicadores de salud.
  • Direcciones — InternalIP y Hostname.
  • Info del sistema — Sistema operativo, Arquitectura, Container Runtime, Versión de Kubelet.
  • Ver Pods en este Node — botón para ver todos los pods ejecutándose en ese node.

La app descarga la kubeconfig de la API de Civo, extrae los certificados y se conecta directamente a la API de Kubernetes usando autenticación por certificado de cliente. No se necesita kubectl ni herramientas externas.

Lista de Pods

Desde la vista de detalle del node, haga clic en Ver Pods en este Node para ver todos los pods en ese node. Cada pod muestra:

  • Badge de estado (Running, Pending, Failed, etc.)
  • Namespace
  • Conteo de contenedores listos
  • Conteo de reinicios

Logs de Pods

Haga clic en un pod en la lista de pods para ver sus logs:

  • Salida de logs desplazable en fuente monoespaciada
  • Botón de auto-scroll — desplaza automáticamente a las entradas de log más recientes
  • Botón de actualizar — recarga los últimos logs
  • Encabezado del pod con nombre, namespace y estado

Red

Redes

Ver y crear redes virtuales. La red predeterminada no se puede eliminar. Haga clic en + para crear una con una etiqueta y CIDR opcional.

Firewalls

Haga clic en un firewall para ver todas sus reglas. Cada regla muestra:

  • Protocolo, rango de puertos, CIDR
  • Dirección (ingress/egress) y acción (permitir/denegar) como badges de colores
  • Etiqueta

Haga clic en + en la vista de detalle de reglas para añadir una nueva regla. Elimine reglas mediante el menú contextual de clic derecho (requiere escribir el nombre de la regla).

Load Balancers

Ver detalles del load balancer incluyendo algoritmo, IPs públicas/privadas, política de tráfico y lista de backends. Eliminar mediante menú contextual.

Dominios & DNS

Expanda un dominio para ver todos los registros DNS. Cada registro muestra tipo, nombre, valor y TTL.

  • Añadir registro — haga clic en el botón «Añadir registro» dentro del dominio.
  • Editar registro — haga clic derecho en un registro y seleccione «Editar».
  • Eliminar registro — haga clic derecho y seleccione «Eliminar».

Almacenamiento & datos

Bases de datos

Haga clic en una base de datos para ver los detalles de conexión (host, puerto, entrada DNS, IP privada), configuración (software, versión, tamaño, nodes) y asociaciones de red/firewall.

Para crear: haga clic en +. Elija PostgreSQL o MySQL, versión, tamaño, nodes, red y firewall.

Volumes

Haga clic en un volume para ver el estado de conexión (a qué instancia o cluster está conectado), punto de montaje, tamaño, estado de arranque, región y fecha de creación.

La lista muestra si cada volume está «conectado a instancia», «conectado a cluster» o «disponible».

Limpiar disponibles — botón en la barra de herramientas para eliminar masivamente todos los volumes con estado «disponible». Requiere escribir DELETE ALL DATA para confirmar.

Object Stores

Ver object stores con endpoint y tamaño máximo. Haga clic derecho para:

  • Mostrar credenciales — muestra el access key ID y el secret access key (seleccionable para copiar).
  • Eliminar — elimina permanentemente el object store.

Cuenta

Regiones

Ver todas las regiones de Civo disponibles. Haga clic en Usar para cambiar su región activa. Todas las llamadas API posteriores se dirigirán a la región seleccionada.

Acerca de

Muestra la versión de la app, número de build, información del desarrollador y enlaces a la Política de privacidad y los Términos de uso.

Eliminar recursos

Todas las operaciones de eliminación requieren que escriba el nombre exacto del recurso para confirmar. Esto evita la eliminación accidental de infraestructura cloud crítica.

Para eliminar: haga clic derecho en un recurso y seleccione Eliminar. Aparece un diálogo de confirmación donde debe escribir el nombre exactamente como se muestra. El botón de eliminar solo se activa cuando el nombre coincide.

Acceso completo

La gestión de firewalls en la barra de menús es gratuita. El dashboard completo con toda la gestión de recursos requiere una compra única de Acceso completo (14,99 $).

  • Restaurar compra — si compró previamente en otro dispositivo.
  • Canjear código — para códigos de oferta de Apple.
  • Compartir en familia — activado.

Consejos

  • Todos los campos de texto con IPs, endpoints y credenciales admiten selección de texto para copiar fácilmente.
  • La app actualiza automáticamente el estado del firewall cada 60 segundos.
  • Las tarjetas de recursos del dashboard son clicables — navegue a cualquier sección con un clic.
  • Las reglas de firewall creadas por esta app están etiquetadas como civo-cloud-<hostname>-<firewall> y solo se gestionan las reglas de esta app.
  • El flujo de detalle de Kubernetes: Lista de Clusters → Detalle del Cluster → Detalle del Node → Lista de Pods → Logs de Pods.
