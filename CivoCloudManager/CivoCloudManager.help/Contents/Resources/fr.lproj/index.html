Aide Civo Cloud Manager

Gérez votre infrastructure Civo Cloud depuis votre Mac.

Premiers pas

Civo Cloud Manager s'exécute comme une app dans la barre de menus. Après le lancement, une icône de bouclier apparaît dans la barre de menus de votre Mac.

  1. Cliquez sur l'icône du bouclier pour ouvrir le popover.
  2. Lors du premier lancement, l'Assistant de configuration s'ouvre automatiquement.
  3. Saisissez votre Civo API Key (disponible sur civo.com → Account → Security → API Keys).
  4. Sélectionnez votre région (ex. fra1, lon1, nyc1).
  5. Choisissez les firewalls à gérer et configurez les ports.
  6. Activez optionnellement le Lancement au démarrage.

Barre de menus

Le popover de la barre de menus offre une gestion rapide des firewalls :

IcôneSignification
Bouclier (vert)Tous les firewalls fermés
Bouclier (jaune)Certains firewalls ouverts pour votre IP
Bouclier (rouge)Configuration non terminée
  • Ouvrir — crée une règle de firewall autorisant votre IP publique actuelle sur le port configuré.
  • Fermer — supprime la règle.
  • Tout ouvrir / Tout fermer — actions groupes pour tous les firewalls gérés.
  • Tableau de bord — ouvre la fenêtre de gestion complète.
  • Réglages — rouvre l'assistant de configuration.
  • Actualiser — vérifie manuellement le statut.

Votre IP publique est détectée automatiquement via ipify.org avec des fournisseurs de secours. L'app ne gère que les règles qu'elle a créées — identifiées par votre nom d'hôte.

Tableau de bord

Le tableau de bord affiche le nombre de ressources et l'utilisation des quotas. Cliquez sur une carte de ressource pour naviguer directement vers cette section.

Jauges de quota

Les jauges circulaires montrent l'utilisation par rapport à la limite pour tous les quotas du compte (RAM et DB RAM affichés en Go).

Demander un changement de quota

Cliquez sur Demander un changement à côté de « Utilisation des quotas » pour ouvrir un formulaire avec des steppers pour toutes les limites de quota. Soumet une demande de modification via la Civo API.

Compute

Instances

Afficher, créer et supprimer des instances de calcul. Cliquez sur une instance pour voir les détails :

  • Spécifications CPU, RAM et disque
  • Adresses IP publiques et privées
  • Mot de passe root initial (bouton afficher/masquer)
  • Tags, région, date de création
  • Associations réseau et firewall

Pour créer : cliquez sur + dans la barre d'outils. Remplissez le hostname, la taille, l'image disque, le réseau, le firewall, la SSH Key et les tags.

SSH Keys

Gérez les SSH Keys pour l'accès aux instances. Pour ajouter une clé : cliquez sur +, saisissez un nom et collez votre clé publique.

Kubernetes

Clusters

Afficher, créer et supprimer des clusters Kubernetes. Cliquez sur un cluster pour voir sa vue détaillée avec :

  • Statistiques — Nodes, Pools, état de santé, applications installées en un coup d'œil.
  • Info du cluster — endpoint API, IP maître, DNS, plugin CNI, taille des nodes, date de création.
  • Conditions — ControlPlaneReady, WorkerNodesReady, ClusterVersionSync avec indicateurs de santé.
  • Node Pools — cliquez pour développer et voir l'ID du pool, la taille, le nombre de nodes, les IPs publiques, les noms d'instance, les labels et les taints.
  • Labels — cliquez sur Modifier pour ajouter, supprimer ou modifier les labels d'un node pool.
  • Applications installées — cert-manager, ingress-nginx, etc.
  • Sauvegarder Kubeconfig — bouton dans la barre d'outils pour exporter la kubeconfig en fichier .yaml.

Pour créer : cliquez sur +. Configurez le nom, le plugin CNI, la taille des nodes, le nombre de nodes, le réseau et les apps du marketplace.

Détails du Node (API K8s)

Cliquez sur le nom d'un node dans un node pool pour vous connecter directement à l'API Kubernetes et voir les informations détaillées du node :

  • Cartes de ressources — CPU, Memory et Pods montrant la capacité vs. les valeurs allouables.
  • Conditions — Ready, MemoryPressure, DiskPressure, PIDPressure avec indicateurs de santé.
  • Adresses — InternalIP et Hostname.
  • Info système — Système d'exploitation, Architecture, Container Runtime, Version Kubelet.
  • Voir les Pods sur ce Node — bouton pour voir tous les pods exécutés sur ce node.

L'app télécharge la kubeconfig depuis l'API Civo, extrait les certificats et se connecte directement à l'API Kubernetes en utilisant l'authentification par certificat client. Aucun kubectl ni outil externe n'est nécessaire.

Liste des Pods

Depuis la vue détaillée du node, cliquez sur Voir les Pods sur ce Node pour voir tous les pods sur ce node. Chaque pod affiche :

  • Badge de statut (Running, Pending, Failed, etc.)
  • Namespace
  • Nombre de conteneurs prêts
  • Nombre de redémarrages

Logs des Pods

Cliquez sur un pod dans la liste des pods pour voir ses logs :

  • Sortie de logs défilable en police monospace
  • Bouton auto-scroll — défile automatiquement vers les dernières entrées de log
  • Bouton actualiser — recharge les derniers logs
  • En-tête du pod avec nom, namespace et statut

Réseau

Réseaux

Afficher et créer des réseaux virtuels. Le réseau par défaut ne peut pas être supprimé. Cliquez sur + pour en créer un avec un libellé et un CIDR optionnel.

Firewalls

Cliquez sur un firewall pour voir toutes ses règles. Chaque règle affiche :

  • Protocole, plage de ports, CIDR
  • Direction (ingress/egress) et action (autoriser/refuser) sous forme de badges colorés
  • Libellé

Cliquez sur + dans la vue détaillée des règles pour ajouter une nouvelle règle. Supprimez les règles via le menu contextuel (clic droit, saisie du nom de la règle requise).

Load Balancers

Affichez les détails du load balancer, y compris l'algorithme, les IPs publiques/privées, la politique de trafic et la liste des backends. Suppression via le menu contextuel.

Domaines & DNS

Développez un domaine pour voir tous les enregistrements DNS. Chaque enregistrement affiche le type, le nom, la valeur et le TTL.

  • Ajouter un enregistrement — cliquez sur le bouton « Ajouter un enregistrement » dans le domaine.
  • Modifier un enregistrement — faites un clic droit sur un enregistrement et sélectionnez « Modifier ».
  • Supprimer un enregistrement — faites un clic droit et sélectionnez « Supprimer ».

Stockage & données

Bases de données

Cliquez sur une base de données pour voir les détails de connexion (host, port, entrée DNS, IP privée), la configuration (logiciel, version, taille, nodes) et les associations réseau/firewall.

Pour créer : cliquez sur +. Choisissez PostgreSQL ou MySQL, la version, la taille, les nodes, le réseau et le firewall.

Volumes

Cliquez sur un volume pour voir le statut de rattachement (à quelle instance ou cluster il est rattaché), le point de montage, la taille, le statut de démarrage, la région et la date de création.

La liste indique si chaque volume est « rattaché à une instance », « rattaché à un cluster » ou « disponible ».

Nettoyer les disponibles — bouton dans la barre d'outils pour supprimer en masse tous les volumes avec le statut « disponible ». Nécessite de saisir DELETE ALL DATA pour confirmer.

Object Stores

Affichez les object stores avec endpoint et taille maximale. Faites un clic droit pour :

  • Afficher les identifiants — affiche l'access key ID et le secret access key (sélectionnable pour copier).
  • Supprimer — supprime définitivement l'object store.

Compte

Régions

Affichez toutes les régions Civo disponibles. Cliquez sur Utiliser pour changer votre région active. Tous les appels API suivants cibleront la région sélectionnée.

À propos

Affiche la version de l'app, le numéro de build, les informations sur le développeur et les liens vers la Politique de confidentialité et les Conditions d'utilisation.

Supprimer des ressources

Toutes les opérations de suppression nécessitent de saisir le nom exact de la ressource pour confirmer. Cela empêche la suppression accidentelle d'infrastructure cloud critique.

Pour supprimer : faites un clic droit sur une ressource et sélectionnez Supprimer. Une fenêtre de confirmation apparaît où vous devez saisir le nom exactement tel qu'il est affiché. Le bouton de suppression ne s'active que lorsque le nom correspond.

Accès complet

La gestion des firewalls dans la barre de menus est gratuite. Le tableau de bord complet avec toute la gestion des ressources nécessite un achat unique d'Accès complet (14,99 $).

  • Restaurer l'achat — si vous avez précédemment acheté sur un autre appareil.
  • Échanger un code — pour les codes promotionnels Apple.
  • Partage familial — activé.

Conseils

  • Tous les champs de texte avec IPs, endpoints et identifiants prennent en charge la sélection de texte pour une copie facile.
  • L'app actualise automatiquement le statut des firewalls toutes les 60 secondes.
  • Les cartes de ressources du tableau de bord sont cliquables — naviguez vers n'importe quelle section en un clic.
  • Les règles de firewall créées par cette app sont étiquetées civo-cloud-<hostname>-<firewall> et seules les règles de cette app sont gérées.
  • Le flux de détail Kubernetes : Liste des Clusters → Détail du Cluster → Détail du Node → Liste des Pods → Logs des Pods.
