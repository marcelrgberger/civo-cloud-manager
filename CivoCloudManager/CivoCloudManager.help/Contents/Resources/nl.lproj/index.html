Civo Cloud Manager Help

Beheer uw Civo Cloud-infrastructuur vanaf uw Mac.

Aan de slag

Civo Cloud Manager draait als een menubalk-app. Na het starten verschijnt een schildpictogram in de menubalk van uw Mac.

  1. Klik op het schildpictogram om de popover te openen.
  2. Bij de eerste keer starten opent de Configuratieassistent automatisch.
  3. Voer uw Civo API Key in (te vinden op civo.com → Account → Security → API Keys).
  4. Selecteer uw regio (bijv. fra1, lon1, nyc1).
  5. Kies welke firewalls u wilt beheren en stel poorten in.
  6. Schakel optioneel Starten bij inloggen in.

Menubalk

De menubalk-popover biedt snel toegang tot firewallbeheer:

PictogramBetekenis
Schild (groen)Alle firewalls gesloten
Schild (geel)Sommige firewalls open voor uw IP
Schild (rood)Configuratie niet voltooid
  • Openen — maakt een firewallregel aan die uw huidige openbare IP toestaat op de geconfigureerde poort.
  • Sluiten — verwijdert de regel.
  • Alles openen / Alles sluiten — bulkacties voor alle beheerde firewalls.
  • Dashboard — opent het volledige beheervenster.
  • Instellingen — opent de configuratieassistent opnieuw.
  • Vernieuwen — controleert de status handmatig.

Uw openbare IP wordt automatisch gedetecteerd via ipify.org met fallback-providers. De app beheert alleen regels die hij zelf heeft aangemaakt — gelabeld met uw hostnaam.

Dashboard

Het Dashboard toont het aantal resources en quotumgebruik. Klik op een resourcekaart om direct naar die sectie te navigeren.

Quotummeters

Cirkelvormige meters tonen het gebruik ten opzichte van de limiet voor alle accountquota (RAM en DB RAM weergegeven in GB).

Quotumwijziging aanvragen

Klik op Wijziging aanvragen naast “Quotumgebruik” om een formulier te openen met steppers voor alle quotumlimieten. Dient een wijzigingsverzoek in via de Civo API.

Compute

Instanties

Bekijk, maak en verwijder compute-instanties. Klik op een instantie om details te zien:

  • CPU-, RAM- en schijfspecificaties
  • Openbare en privé-IP-adressen
  • Initiëel root-wachtwoord (toon/verberg-schakelaar)
  • Tags, regio, aanmaakdatum
  • Netwerk- en firewallassociaties

Om te maken: klik op + in de werkbalk. Vul hostnaam, grootte, schijfimage, netwerk, firewall, SSH Key en tags in.

SSH Keys

Beheer SSH Keys voor toegang tot instanties. Om een sleutel toe te voegen: klik op +, voer een naam in en plak uw openbare sleutel.

Kubernetes

Clusters

Bekijk, maak en verwijder Kubernetes-clusters. Klik op een cluster om de detailweergave te zien met:

  • Statistieken — Nodes, Pools, gezondheidsstatus, geïnstalleerde apps in één oogopslag.
  • Clusterinfo — API-endpoint, master-IP, DNS, CNI-plugin, node-grootte, aanmaakdatum.
  • Condities — ControlPlaneReady, WorkerNodesReady, ClusterVersionSync met gezondheidsindicatoren.
  • Node Pools — klik om uit te vouwen en pool-ID, grootte, aantal nodes, openbare IP's, instantienamen, labels en taints te zien.
  • Labels — klik op Bewerken om labels van een node pool toe te voegen, te verwijderen of te wijzigen.
  • Geïnstalleerde applicaties — cert-manager, ingress-nginx, enz.
  • Kubeconfig opslaan — werkbalkknop om de kubeconfig te exporteren als .yaml-bestand.

Om te maken: klik op +. Configureer naam, CNI-plugin, node-grootte, aantal nodes, netwerk en marketplace-apps.

Node-details (K8s API)

Klik op een node-naam in een node pool om rechtstreeks verbinding te maken met de Kubernetes API en gedetailleerde node-informatie te zien:

  • Resourcekaarten — CPU, Memory en Pods met capaciteit vs. toewijsbare waarden.
  • Condities — Ready, MemoryPressure, DiskPressure, PIDPressure met gezondheidsindicatoren.
  • Adressen — InternalIP en Hostname.
  • Systeeminfo — Besturingssysteem, Architectuur, Container Runtime, Kubelet-versie.
  • Pods op deze Node bekijken — knop om alle pods te zien die op die node draaien.

De app downloadt de kubeconfig van de Civo API, extraheert de certificaten en maakt rechtstreeks verbinding met de Kubernetes API via clientcertificaat-authenticatie. Geen kubectl of externe tools nodig.

Podlijst

Klik vanuit de node-detailweergave op Pods op deze Node bekijken om alle pods op die node te zien. Elke pod toont:

  • Statusbadge (Running, Pending, Failed, enz.)
  • Namespace
  • Aantal gereedstaande containers
  • Aantal herstartten

Pod-logs

Klik op een pod in de podlijst om de logs te bekijken:

  • Scrollbare loguitvoer in monospace-lettertype
  • Auto-scroll schakelaar — scrollt automatisch naar de nieuwste logvermeldingen
  • Vernieuwen-knop — herlaadt de nieuwste logs
  • Pod-header met naam, namespace en status

Netwerk

Netwerken

Bekijk en maak virtuele netwerken. Het standaardnetwerk kan niet worden verwijderd. Klik op + om er een te maken met een label en optioneel CIDR.

Firewalls

Klik op een firewall om alle regels te zien. Elke regel toont:

  • Protocol, poortbereik, CIDR
  • Richting (ingress/egress) en actie (toestaan/weigeren) als gekleurde badges
  • Label

Klik op + in de regeldetailweergave om een nieuwe regel toe te voegen. Verwijder regels via het contextmenu (rechtsklik, vereist het typen van de regelnaam).

Load Balancers

Bekijk load balancer-details inclusief algoritme, openbare/privé-IP's, verkeersbeleid en backendlijst. Verwijderen via contextmenu.

Domeinen & DNS

Vouw een domein uit om alle DNS-records te zien. Elk record toont type, naam, waarde en TTL.

  • Record toevoegen — klik op de knop “Record toevoegen” binnen het domein.
  • Record bewerken — klik rechts op een record en selecteer “Bewerken”.
  • Record verwijderen — klik rechts en selecteer “Verwijderen”.

Opslag & gegevens

Databases

Klik op een database om verbindingsdetails te zien (host, poort, DNS-vermelding, privé-IP), configuratie (software, versie, grootte, nodes) en netwerk-/firewallassociaties.

Om te maken: klik op +. Kies PostgreSQL of MySQL, versie, grootte, nodes, netwerk en firewall.

Volumes

Klik op een volume om de koppelingsstatus te zien (aan welke instantie of cluster het is gekoppeld), mountpoint, grootte, opstartstatus, regio en aanmaakdatum.

De lijst toont of elk volume “gekoppeld aan instantie”, “gekoppeld aan cluster” of “beschikbaar” is.

Beschikbare opruimen — werkbalkknop om alle volumes met de status “beschikbaar” in bulk te verwijderen. Vereist het typen van DELETE ALL DATA om te bevestigen.

Object Stores

Bekijk object stores met endpoint en maximale grootte. Klik rechts voor:

  • Referenties tonen — toont access key ID en secret access key (selecteerbaar om te kopiëren).
  • Verwijderen — verwijdert de object store permanent.

Account

Regio's

Bekijk alle beschikbare Civo-regio's. Klik op Gebruiken om uw actieve regio te wijzigen. Alle volgende API-aanroepen worden gericht op de geselecteerde regio.

Over

Toont de app-versie, buildnummer, ontwikkelaarsinformatie en links naar het Privacybeleid en de Gebruiksvoorwaarden.

Resources verwijderen

Alle verwijderingsacties vereisen dat u de exacte resourcenaam typt ter bevestiging. Dit voorkomt het per ongeluk verwijderen van kritieke cloudinfrastructuur.

Om te verwijderen: klik rechts op een resource en selecteer Verwijderen. Er verschijnt een bevestigingsvenster waarin u de naam exact moet typen zoals weergegeven. De verwijderknop wordt pas actief wanneer de naam overeenkomt.

Volledige toegang

Het firewallbeheer via de menubalk is gratis. Het volledige dashboard met al het resourcebeheer vereist een eenmalige aankoop van Volledige toegang ($ 14,99).

  • Aankoop herstellen — als u eerder op een ander apparaat hebt gekocht.
  • Code inwisselen — voor Apple-aanbiedingscodes.
  • Gezinsdeling — ingeschakeld.

Tips

  • Alle tekstvelden met IP's, endpoints en referenties ondersteunen tekstselectie voor eenvoudig kopiëren.
  • De app vernieuwt de firewallstatus automatisch elke 60 seconden.
  • Dashboard-resourcekaarten zijn klikbaar — navigeer met één klik naar elke sectie.
  • Firewallregels die door deze app zijn aangemaakt, zijn gelabeld als civo-cloud-<hostname>-<firewall> en alleen de regels van deze app worden beheerd.
  • De Kubernetes-drill-downstroom: Clusterlijst → Clusterdetail → Nodedetail → Podlijst → Podlogs.
