Pomoc Civo Cloud Manager

Zarządzaj infrastrukturą Civo Cloud ze swojego Maca.

Pierwsze kroki

Civo Cloud Manager działa jako aplikacja w pasku menu. Po uruchomieniu w pasku menu Maca pojawia się ikona tarczy.

  1. Kliknij ikonę tarczy, aby otworzyć popover.
  2. Przy pierwszym uruchomieniu Kreator konfiguracji otwiera się automatycznie.
  3. Wprowadź swój Civo API Key (znajdziesz go na civo.com → Account → Security → API Keys).
  4. Wybierz swój region (np. fra1, lon1, nyc1).
  5. Wybierz, które firewalle chcesz zarządzać i ustaw porty.
  6. Opcjonalnie włącz Uruchamianie przy logowaniu.

Pasek menu

Popover paska menu zapewnia szybki dostęp do zarządzania firewallami:

IkonaZnaczenie
Tarcza (zielona)Wszystkie firewalle zamknięte
Tarcza (żółta)Niektóre firewalle otwarte dla Twojego IP
Tarcza (czerwona)Konfiguracja niezakończona
  • Otwórz — tworzy regułę firewalla zezwalającą na Twój aktualny publiczny IP na skonfigurowanym porcie.
  • Zamknij — usuwa regułę.
  • Otwórz wszystkie / Zamknij wszystkie — akcje masowe dla wszystkich zarządzanych firewalli.
  • Dashboard — otwiera pełne okno zarządzania.
  • Ustawienia — ponownie otwiera kreator konfiguracji.
  • Odśwież — ręcznie sprawdza status ponownie.

Twój publiczny IP jest automatycznie wykrywany przez ipify.org z zapasowymi dostawcami. Aplikacja zarządza tylko regułami, które sama utworzyła — oznaczonymi Twoją nazwą hosta.

Dashboard

Dashboard pokazuje liczbę zasobów i wykorzystanie limitów. Kliknij dowolną kartę zasobu, aby przejść bezpośrednio do tej sekcji.

Wskaźniki limitów

Okrągłe wskaźniki pokazują wykorzystanie w stosunku do limitu dla wszystkich limitów konta (RAM i DB RAM wyświetlane w GB).

Prośba o zmianę limitu

Kliknij Prośba o zmianę obok „Wykorzystanie limitów”, aby otworzyć formularz ze stepperami dla wszystkich limitów. Wysyła prośbę o zmianę przez Civo API.

Compute

Instancje

Przeglądaj, tworzaj i usuwaj instancje obliczeniowe. Kliknij instancję, aby zobaczyć szczegóły:

  • Specyfikacje CPU, RAM i dysku
  • Publiczne i prywatne adresy IP
  • Początkowe hasło root (przełącznik pokaż/ukryj)
  • Tagi, region, data utworzenia
  • Powiązania sieciowe i firewallowe

Aby utworzyć: kliknij + na pasku narzędzi. Wypełnij nazwę hosta, rozmiar, obraz dysku, sieć, firewall, SSH Key i tagi.

SSH Keys

Zarządzaj SSH Keys do dostępu do instancji. Aby dodać klucz: kliknij +, wprowadź nazwę i wklej swój klucz publiczny.

Kubernetes

Klastry

Przeglądaj, tworzaj i usuwaj klastry Kubernetes. Kliknij klaster, aby zobaczyć widok szczegółowy z:

  • Statystyki — Nodes, Pools, stan zdrowia, zainstalowane aplikacje w jednym widoku.
  • Info o klastrze — endpoint API, IP mastera, DNS, plugin CNI, rozmiar node, data utworzenia.
  • Warunki — ControlPlaneReady, WorkerNodesReady, ClusterVersionSync ze wskaźnikami zdrowia.
  • Node Pools — kliknij, aby rozwinąć i zobaczyć ID puli, rozmiar, liczbę node'ów, publiczne IP, nazwy instancji, labels i taints.
  • Labels — kliknij Edytuj, aby dodawać, usuwać lub modyfikować labels w node pool.
  • Zainstalowane aplikacje — cert-manager, ingress-nginx itp.
  • Zapisz Kubeconfig — przycisk na pasku narzędzi do eksportu kubeconfig jako plik .yaml.

Aby utworzyć: kliknij +. Skonfiguruj nazwę, plugin CNI, rozmiar node, liczbę node'ów, sieć i aplikacje z marketplace.

Połącz z API Kubernetes

W widoku szczegółów klastra kliknij Połącz z API Kubernetes, aby uzyskać dostęp do danych na żywo bezpośrednio z API Kubernetes:

  • Wskaźniki CPU i pamięci — okrągłe wskaźniki procentowe pokazujące wykorzystanie zasobów klastra w czasie rzeczywistym (wymaga metrics-server zainstalowanego na klastrze).
  • Liczba Podów i zdrowie Node'ów — wskaźniki na żywo stanu klastra.
  • Ostatnie zdarzenia — zdarzenia klastra z ostrzeżeniami podświetlonymi na pomarańczowo.
  • Deployments — obciążenia robocze pokazujące gotowe vs. pożądane repliki.

Jeśli metrics-server nie jest zainstalowany na klastrze, aplikacja wyświetla statyczne statystyki klastra z API Civo.

Podczas łączenia z API Kubernetes aplikacja automatycznie otwiera port 6443 w firewallu klastra dla Twojego aktualnego IP. Reguła jest oznaczona jako civo-cloud-<hostname>-k8s-api i jest automatycznie usuwana po powrócie z widoku klastra.

Szczegóły Node (K8s API)

Kliknij nazwę node'a w node pool, aby połączyć się bezpośrednio z API Kubernetes i zobaczyć szczegółowe informacje o node:

  • Karty zasobów — CPU, Memory i Pods pokazujące pojemność vs. wartości przydzielalne.
  • Warunki — Ready, MemoryPressure, DiskPressure, PIDPressure ze wskaźnikami zdrowia.
  • Adresy — InternalIP i Hostname.
  • Info o systemie — System operacyjny, Architektura, Container Runtime, Wersja Kubelet.
  • Pokaż Pody na tym Node — przycisk do wyświetlenia wszystkich podów działających na tym node.

Aplikacja pobiera kubeconfig z API Civo, wyodrębnia certyfikaty i łączy się bezpośrednio z API Kubernetes przy użyciu uwierzytelniania certyfikatem klienta. Nie jest potrzebny kubectl ani żadne zewnętrzne narzędzia.

Lista Podów

W widoku szczegółów node'a kliknij Pokaż Pody na tym Node, aby zobaczyć wszystkie pody na tym node. Każdy pod pokazuje:

  • Badge statusu (Running, Pending, Failed itp.)
  • Namespace
  • Liczbę gotowych kontenerów
  • Liczbę restartów

Logi Podów

Kliknij pod w liście podów, aby zobaczyć jego logi:

  • Przewijalna wyjście logów w czcionce monospace
  • Przełącznik auto-scroll — automatycznie przewija do najnowszych wpisów logów
  • Przycisk odśwież — przeladowuje najnowsze logi
  • Nagłówek poda z nazwą, namespace i statusem

Sieć

Sieci

Przeglądaj i tworzaj sieci wirtualne. Domyślna sieć nie może być usunięta. Kliknij +, aby utworzyć sieć z etykietą i opcjonalnym CIDR.

Firewalle

Kliknij firewall, aby zobaczyć wszystkie jego reguły. Każda reguła pokazuje:

  • Protokół, zakres portów, CIDR
  • Kierunek (ingress/egress) i akcja (zezwól/odmowa) jako kolorowe badge
  • Etykieta

Kliknij + w widoku szczegółów reguł, aby dodać nową regułę. Usuwaj reguły przez menu kontekstowe (prawym przyciskiem, wymaga wpisania nazwy reguły).

Load Balancery

Przeglądaj szczegóły load balancera, w tym algorytm, publiczne/prywatne IP, politykę ruchu i listę backendów. Usuwanie przez menu kontekstowe.

Domeny & DNS

Rozwiń domenę, aby zobaczyć wszystkie rekordy DNS. Każdy rekord pokazuje typ, nazwę, wartość i TTL.

  • Dodaj rekord — kliknij przycisk „Dodaj rekord” wewnątrz domeny.
  • Edytuj rekord — kliknij prawym przyciskiem na rekordzie i wybierz „Edytuj”.
  • Usuń rekord — kliknij prawym przyciskiem i wybierz „Usuń”.

Magazyn & dane

Bazy danych

Kliknij bazę danych, aby zobaczyć szczegóły połączenia (host, port, wpis DNS, prywatne IP), konfigurację (oprogramowanie, wersja, rozmiar, nodes) oraz powiązania sieciowe/firewallowe.

Aby utworzyć: kliknij +. Wybierz PostgreSQL lub MySQL, wersję, rozmiar, nodes, sieć i firewall.

Wolumeny

Kliknij wolumen, aby zobaczyć status podłączenia (do której instancji lub klastra jest podłączony), punkt montowania, rozmiar, status rozruchowy, region i datę utworzenia.

Lista pokazuje, czy każdy wolumen jest „podłączony do instancji”, „podłączony do klastra” lub „dostępny”.

Wyczyść dostępne — przycisk na pasku narzędzi do masowego usuwania wszystkich wolumenów ze statusem „dostępny”. Wymaga wpisania DELETE ALL DATA w celu potwierdzenia.

Object Stores

Kliknij object store, aby otworzyć jego widok szczegółowy pokazujący:

  • Dane uwierzytelniające — endpoint, URL bucketa, access key i secret key (wszystkie do zaznaczenia i skopiowania).
  • Konfiguracja — maksymalny rozmiar, region, data utworzenia i status.
  • Zmień rozmiar — stepper do zmiany maksymalnego rozmiaru. Zmiany są wysyłane przez PUT do API Civo.

Kliknij prawym przyciskiem na object store w liście, aby go usunąć.

Dane uwierzytelniające (access key i secret key) pochodzą bezpośrednio z zasobu object store, a nie z oddzielnego endpointu.

Konto

Regiony

Przeglądaj wszystkie dostępne regiony Civo. Kliknij Użyj, aby zmienić aktywny region. Wszystkie kolejne wywołania API będą kierowane do wybranego regionu.

O aplikacji

Pokazuje wersję aplikacji, numer buildu, informacje o deweloperze oraz linki do Polityki prywatności i Warunków użytkowania.

Usuwanie zasobów

Wszystkie operacje usuwania wymagają wpisania dokładnej nazwy zasobu w celu potwierdzenia. Zapobiega to przypadkowemu usunięciu krytycznej infrastruktury chmurowej.

Aby usunąć: kliknij prawym przyciskiem na zasobie i wybierz Usuń. Pojawia się okno potwierdzenia, w którym musisz wpisać nazwę dokładnie tak, jak jest wyświetlana. Przycisk usuwania staje się aktywny dopiero po dopasowaniu nazwy.

Pełny dostęp

Zarządzanie firewallami w pasku menu jest bezpłatne. Pełny dashboard z całym zarządzaniem zasobami wymaga jednorazowego zakupu Pełnego dostępu (14,99 $).

  • Przywróć zakup — jeśli wcześniej kupiono na innym urządzeniu.
  • Zrealizuj kod — dla kodów promocyjnych Apple.
  • Udostępnianie rodzinne — włączone.

Wskazówki

  • Wszystkie pola tekstowe z adresami IP, endpointami i danymi uwierzytelniającymi obsługują zaznaczanie tekstu do łatwego kopiowania.
  • Aplikacja automatycznie odświeża status firewalla co 60 sekund.
  • Karty zasobów na dashboardzie są klikalne — przejdź do dowolnej sekcji jednym kliknięciem.
  • Reguły firewalla utworzone przez tę aplikację są oznaczone jako civo-cloud-<hostname>-<firewall> i tylko reguły tej aplikacji są zarządzane.
  • Przepływ szczegółów Kubernetes: Lista klastrów → Szczegóły klastra → Szczegóły Node → Lista Podów → Logi Podów.
  • Podczas łączenia z API Kubernetes port 6443 jest automatycznie otwierany i zamykany dla Twojego IP — nie jest potrzebna ręczna konfiguracja firewalla.
