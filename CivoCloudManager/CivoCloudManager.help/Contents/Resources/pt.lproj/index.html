Ajuda do Civo Cloud Manager

Gerencie sua infraestrutura Civo Cloud a partir do seu Mac.

Primeiros passos

O Civo Cloud Manager funciona como um app na barra de menus. Após a inicialização, um ícone de escudo aparece na barra de menus do seu Mac.

  1. Clique no ícone do escudo para abrir o popover.
  2. Na primeira execução, o Assistente de configuração abre automaticamente.
  3. Insira sua Civo API Key (encontre-a em civo.com → Account → Security → API Keys).
  4. Selecione sua região (ex.: fra1, lon1, nyc1).
  5. Escolha quais firewalls gerenciar e configure as portas.
  6. Opcionalmente, ative Iniciar ao fazer login.

Barra de menus

O popover da barra de menus oferece gerenciamento rápido de firewalls:

ÍconeSignificado
Escudo (verde)Todos os firewalls fechados
Escudo (amarelo)Alguns firewalls abertos para seu IP
Escudo (vermelho)Configuração não concluída
  • Abrir — cria uma regra de firewall permitindo seu IP público atual na porta configurada.
  • Fechar — remove a regra.
  • Abrir todos / Fechar todos — ações em massa para todos os firewalls gerenciados.
  • Dashboard — abre a janela de gerenciamento completo.
  • Configurações — reabre o assistente de configuração.
  • Atualizar — verifica o status manualmente.

Seu IP público é detectado automaticamente via ipify.org com provedores de fallback. O app gerencia apenas as regras que ele mesmo criou — identificadas com seu hostname.

Dashboard

O Dashboard mostra a contagem de recursos e o uso de cotas. Clique em qualquer cartão de recurso para navegar diretamente até essa seção.

Indicadores de cota

Indicadores circulares mostram o uso em relação ao limite para todas as cotas da conta (RAM e DB RAM exibidos em GB).

Solicitar alteração de cota

Clique em Solicitar alteração ao lado de “Uso de cota” para abrir um formulário com steppers para todos os limites de cota. Envia uma solicitação de alteração através da Civo API.

Compute

Instâncias

Visualize, crie e exclua instâncias de computação. Clique em uma instância para ver os detalhes:

  • Especificações de CPU, RAM e disco
  • Endereços IP públicos e privados
  • Senha root inicial (botão mostrar/ocultar)
  • Tags, região, data de criação
  • Associações de rede e firewall

Para criar: clique em + na barra de ferramentas. Preencha o hostname, tamanho, imagem de disco, rede, firewall, SSH Key e tags.

SSH Keys

Gerencie SSH Keys para acesso às instâncias. Para adicionar uma chave: clique em +, insira um nome e cole sua chave pública.

Kubernetes

Clusters

Visualize, crie e exclua clusters Kubernetes. Clique em um cluster para ver a visualização detalhada com:

  • Estatísticas — Nodes, Pools, estado de saúde, aplicativos instalados em um relance.
  • Info do cluster — endpoint da API, IP master, DNS, plugin CNI, tamanho do node, data de criação.
  • Condições — ControlPlaneReady, WorkerNodesReady, ClusterVersionSync com indicadores de saúde.
  • Node Pools — clique para expandir e ver ID do pool, tamanho, número de nodes, IPs públicos, nomes de instância, labels e taints.
  • Labels — clique em Editar para adicionar, remover ou modificar labels de um node pool.
  • Aplicativos instalados — cert-manager, ingress-nginx, etc.
  • Salvar Kubeconfig — botão na barra de ferramentas para exportar a kubeconfig como arquivo .yaml.

Para criar: clique em +. Configure nome, plugin CNI, tamanho do node, número de nodes, rede e apps do marketplace.

Detalhes do Node (API K8s)

Clique no nome de um node em um node pool para conectar-se diretamente à API Kubernetes e ver informações detalhadas do node:

  • Cartões de recursos — CPU, Memory e Pods mostrando capacidade vs. valores alocáveis.
  • Condições — Ready, MemoryPressure, DiskPressure, PIDPressure com indicadores de saúde.
  • Endereços — InternalIP e Hostname.
  • Info do sistema — Sistema operacional, Arquitetura, Container Runtime, Versão do Kubelet.
  • Ver Pods neste Node — botão para ver todos os pods em execução nesse node.

O app baixa a kubeconfig da API Civo, extrai os certificados e conecta-se diretamente à API Kubernetes usando autenticação por certificado de cliente. Não é necessário kubectl nem ferramentas externas.

Lista de Pods

Na visualização de detalhes do node, clique em Ver Pods neste Node para ver todos os pods nesse node. Cada pod mostra:

  • Badge de status (Running, Pending, Failed, etc.)
  • Namespace
  • Contagem de containers prontos
  • Contagem de reinicializações

Logs de Pods

Clique em um pod na lista de pods para ver seus logs:

  • Saída de logs rolável em fonte monoespaçada
  • Botão auto-scroll — rola automaticamente para as entradas de log mais recentes
  • Botão atualizar — recarrega os logs mais recentes
  • Cabeçalho do pod com nome, namespace e status

Rede

Redes

Visualize e crie redes virtuais. A rede padrão não pode ser excluída. Clique em + para criar uma com um rótulo e CIDR opcional.

Firewalls

Clique em um firewall para ver todas as suas regras. Cada regra mostra:

  • Protocolo, intervalo de portas, CIDR
  • Direção (ingress/egress) e ação (permitir/negar) como badges coloridos
  • Rótulo

Clique em + na visualização de detalhes das regras para adicionar uma nova regra. Exclua regras pelo menu de contexto (clique direito, requer digitar o nome da regra).

Load Balancers

Visualize detalhes do load balancer, incluindo algoritmo, IPs públicos/privados, política de tráfego e lista de backends. Exclusão pelo menu de contexto.

Domínios & DNS

Expanda um domínio para ver todos os registros DNS. Cada registro mostra tipo, nome, valor e TTL.

  • Adicionar registro — clique no botão “Adicionar registro” dentro do domínio.
  • Editar registro — clique com o botão direito em um registro e selecione “Editar”.
  • Excluir registro — clique com o botão direito e selecione “Excluir”.

Armazenamento & dados

Bancos de dados

Clique em um banco de dados para ver detalhes de conexão (host, porta, entrada DNS, IP privado), configuração (software, versão, tamanho, nodes) e associações de rede/firewall.

Para criar: clique em +. Escolha PostgreSQL ou MySQL, versão, tamanho, nodes, rede e firewall.

Volumes

Clique em um volume para ver o status de conexão (a qual instância ou cluster está conectado), ponto de montagem, tamanho, status de inicialização, região e data de criação.

A lista mostra se cada volume está “conectado a instância”, “conectado a cluster” ou “disponível”.

Limpar disponíveis — botão na barra de ferramentas para excluir em massa todos os volumes com status “disponível”. Requer digitar DELETE ALL DATA para confirmar.

Object Stores

Visualize object stores com endpoint e tamanho máximo. Clique com o botão direito para:

  • Mostrar credenciais — exibe o access key ID e o secret access key (selecionável para copiar).
  • Excluir — remove permanentemente o object store.

Conta

Regiões

Visualize todas as regiões Civo disponíveis. Clique em Usar para mudar sua região ativa. Todas as chamadas API subsequentes serão direcionadas à região selecionada.

Sobre

Mostra a versão do app, número de build, informações do desenvolvedor e links para a Política de Privacidade e os Termos de Uso.

Excluir recursos

Todas as operações de exclusão exigem que você digite o nome exato do recurso para confirmar. Isso evita a exclusão acidental de infraestrutura cloud crítica.

Para excluir: clique com o botão direito em um recurso e selecione Excluir. Uma janela de confirmação aparece onde você deve digitar o nome exatamente como mostrado. O botão excluir só fica ativo quando o nome corresponde.

Acesso completo

O gerenciamento de firewalls na barra de menus é gratuito. O dashboard completo com todo o gerenciamento de recursos requer uma compra única de Acesso completo (US$ 14,99).

  • Restaurar compra — se você comprou anteriormente em outro dispositivo.
  • Resgatar código — para códigos de oferta Apple.
  • Compartilhamento familiar — ativado.

Dicas

  • Todos os campos de texto com IPs, endpoints e credenciais suportam seleção de texto para cópia fácil.
  • O app atualiza automaticamente o status do firewall a cada 60 segundos.
  • Os cartões de recursos do dashboard são clicáveis — navegue para qualquer seção com um clique.
  • As regras de firewall criadas por este app são rotuladas como civo-cloud-<hostname>-<firewall> e apenas as regras deste app são gerenciadas.
  • O fluxo de detalhes Kubernetes: Lista de Clusters → Detalhe do Cluster → Detalhe do Node → Lista de Pods → Logs de Pods.
